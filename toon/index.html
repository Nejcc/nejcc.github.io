Understanding TOON

üìù What is TOON?

TOON (Token-Oriented Object Notation) is a compact data format that combines YAML's indentation-based structure with CSV-style tabular layouts. It's specifically designed to minimize token usage for LLM applications while maintaining human readability.

üéØ Key Features

  • ‚úì Tabular Arrays: Declare fields once, stream data compactly
  • ‚úì Array Length: Explicit [N] declarations for validation
  • ‚úì Field Names: Declared once in {field1,field2} format
  • ‚úì Comma-Separated: Compact row format like CSV

üí° Format Comparison

JSON (Verbose):

{
  "users": [
    {"id": 1, "name": "Alice"},
    {"id": 2, "name": "Bob"}
  ]
}

YAML (Better):

users:
  - id: 1
    name: Alice
  - id: 2
    name: Bob

TOON (Best):

users[2]{id,name}:
  1,Alice
  2,Bob

The [2] declares array length, {id,name,role} declares fields once, and each row is compact comma-separated values.

üîç Query Builder

Convert Query Builder results with chunked processing

PHP Code

use LaravelPlus\Toon\Facades\Toon;
use LaravelPlus\Toon\Database\QueryBuilderToonConverter;

// Basic query
$toon = Toon::fromQueryBuilder(
    User::query()->where('active', true)
);

// With specific columns
$toon = Toon::fromQueryBuilder(
    User::query(),
    ['id', 'name', 'email']
);

// Chunked processing for large datasets
$chunks = [];
foreach (QueryBuilderToonConverter::toToonChunked(
    User::query(),
    chunkSize: 1000
) as $chunkToon) {
    $chunks[] = $chunkToon;
}

// Process chunks using array_map
$processed = array_map(
    fn($chunk) => [
        'toon' => $chunk,
        'count' => count(Toon::decode($chunk)[array_key_first(Toon::decode($chunk))] ?? []),
    ],
    $chunks
);

// Aggregate using array_reduce
$total = array_reduce(
    $processed,
    fn($carry, $chunk) => $carry + $chunk['count'],
    0
);

üîå Dependency Injection with Interfaces

Use dependency injection with TOON interfaces for better testability

use LaravelPlus\Toon\Contracts\ToonManagerInterface;

class DataExportService
{
    public function __construct(
        private ToonManagerInterface $toon
    ) {}

    public function exportUsers(array $userIds): string
    {
        $users = User::whereIn('id', $userIds)->get();
        
        return $this->toon->fromEloquentCollection($users);
    }

    public function exportMultiple(array $collections): array
    {
        // Use array_map instead of foreach
        return array_map(
            fn($collection) => $this->toon->fromEloquentCollection($collection),
            $collections
        );
    }
}

// In a controller or service
class UserController
{
    public function __construct(
        private DataExportService $exportService
    ) {}

    public function export(Request $request): Response
    {
        $userIds = $request->input('user_ids', []);
        $toon = $this->exportService->exportUsers($userIds);
        
        return response($toon, 200)
            ->header('Content-Type', 'text/plain');
    }
}

üîÑ Data Transformation Pipeline

Create transformation pipelines using array methods

PHP Code

use LaravelPlus\Toon\Facades\Toon;

// Raw data from multiple sources
$rawData = [
    'csv' => "id,name,email\n1,Alice,alice@example.com\n2,Bob,bob@example.com",
    'json' => '{"products":[{"id":1,"name":"Widget"},{"id":2,"name":"Gadget"}]}',
    'xml' => '<' . '?xml version="1.0"?>1Charlie',
];

// Transform pipeline using array methods
$transformed = array_reduce(
    array_keys($rawData),
    function ($carry, $format) use ($rawData) {
        $data = match ($format) {
            'csv' => Toon::fromCsv($rawData[$format]),
            'json' => Toon::fromJson($rawData[$format]),
            'xml' => Toon::fromXml($rawData[$format]),
            default => null,
        };
        
        return $data ? [
            ...$carry,
            $format => Toon::decode($data),
        ] : $carry;
    },
    []
);

// Normalize and merge data
$normalized = array_map(
    fn($data, $source) => [
        'source' => $source,
        'records' => array_map(
            fn($record) => array_merge($record, ['source' => $source]),
            is_array($data) && isset($data[array_key_first($data)]) 
                ? $data[array_key_first($data)] 
                : []
        ),
    ],
    $transformed,
    array_keys($transformed)
);

// Filter and aggregate
$aggregated = array_reduce(
    $normalized,
    fn($carry, $item) => [
        ...$carry,
        'records' => array_merge($carry['records'] ?? [], $item['records']),
    ],
    ['sources' => array_keys($transformed)]
);

$finalToon = Toon::encode($aggregated);

Output

sources[3]: csv,json,xml
records[1]{id,name,source}:
  1,Alice,csv

use LaravelPlus\Toon\Facades\Toon;

            ...$carry,

                'content' => $content,
                'size' => strlen($content),
                'mime' => match ($format) {
                    'json' => 'application/json',
                    'xml' => 'application/xml',
                    'yaml' => 'text/yaml',
                    'csv' => 'text/csv',
                    'xlsx' => 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet',
                },
            ],
        ] : $carry;
    },
    []
);

// Get format statistics
$stats = array_map(
    fn($format, $data) => [
        'format' => $format,
        'size' => $data['size'],
        'efficiency' => round((1 - $data['size'] / $exports['json']['size']) * 100, 1),
    ],
    array_keys($exports),
    $exports
);

// Sort by efficiency
usort($stats, fn($a, $b) => $b['efficiency'] <=> $a['efficiency']);
